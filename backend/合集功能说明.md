# 合集功能实现说明

## 后端已完成

### 1. API 修改
- ✅ `/generate_note` 接口添加 `process_playlist` 参数（默认 `false`）
- ✅ 支持处理 B 站视频合集

### 2. 功能特性
- ✅ 自动检测是否为合集
- ✅ 下载合集中的所有视频
- ✅ 分别转写每个视频
- ✅ 分别生成每个视频的笔记
- ✅ 合并所有笔记，以视频标题作为章节标题

### 3. 输出格式
```markdown
# 第一集标题

第一集的笔记内容...

# 第二集标题

第二集的笔记内容...

# 第三集标题

第三集的笔记内容...
```

## 前端需要修改

### 1. 添加UI选项

在视频输入表单中添加一个复选框：

```jsx
<Checkbox 
  checked={processPlaylist}
  onChange={(e) => setProcessPlaylist(e.target.checked)}
>
  处理合集（如果是合集视频，将下载并整合所有视频的笔记）
</Checkbox>
```

### 2. 修改API调用

在调用 `/generate_note` 时，添加 `process_playlist` 参数：

```javascript
const response = await fetch('/api/generate_note', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    video_url: videoUrl,
    platform: platform,
    quality: quality,
    model_name: modelName,
    provider_id: providerId,
    process_playlist: processPlaylist,  // 新增参数
    // ... 其他参数
  }),
});
```

### 3. 状态提示优化

合集处理时间较长，建议优化状态提示：

```javascript
// 根据状态显示不同的提示
if (status === 'downloading') {
  if (processPlaylist) {
    return '正在下载合集...';
  }
  return '正在下载视频...';
}

if (status === 'transcribing') {
  if (processPlaylist) {
    return '正在转写合集视频...（可能需要较长时间）';
  }
  return '正在转写音频...';
}
```

## 使用示例

```json
  "video_url": "https://www.bilibili.com/video/BV1xx411c7mD",
  "platform": "bilibili",
  "process_playlist": false
}
```

### 合集视频
```json
{
  "video_url": "https://www.bilibili.com/video/BV1xx411c7mD",
  "platform": "bilibili",
  "process_playlist": true
}
```

## 注意事项

1. **处理时间**：合集视频处理时间 = 单个视频时间 × 视频数量
2. **存储空间**：会下载所有视频的音频文件
3. **建议**：
   - 默认不勾选（避免意外处理大量视频）
   - 合集视频数量较多时，给用户明确提示
   - 考虑添加进度条显示当前处理第几个视频

## 测试

### 测试用例
1. 单个视频 + 不勾选合集 → 正常处理单个视频
2. 单个视频 + 勾选合集 → 正常处理单个视频（自动降级）
3. 合集视频 + 不勾选合集 → 只处理第一个视频
4. 合集视频 + 勾选合集 → 处理所有视频并合并
